# Roadmap

这个路线图用于记录项目的维护重点。具体优先级会根据平台变化、用户反馈和 provider 稳定性调整。

## 近期

- 建立 provider 兼容性记录，标明每个厂商对小红书公开 URL、抖音下载上传、文件大小限制的支持情况。
- 补充常见错误码和排查路径，尤其是鉴权失败、额度不足、跨域和媒体下载失败。
- 改进采集日志的可读性，方便用户在 issue 中提交脱敏日志。
- 为 OpenAI Audio、DashScope、豆包语音等常用 provider 补充更清晰的配置截图或说明。

## 中期

- 增强小红书和抖音页面提取策略，降低平台页面改版带来的失效率。
- 优化 Markdown 输出模板，支持更稳定的标题、来源、时间、标签和正文结构。
- 增加 provider 适配层的手动测试清单，降低新增厂商时的回归风险。
- 研究更好的大文件处理方式，尤其是抖音长视频和腾讯云 5MB 直传限制。

## 暂不计划

- 内置公共 API Key 或托管转写服务。
- 上传用户内容到项目作者的服务器。
- 绕过平台访问权限或处理用户无权使用的内容。
- 引入重型前端框架和构建流程。

## 需要社区反馈

- 你最常使用哪个转写 provider？
- 哪个平台最容易提取失败？
- Markdown / Obsidian 导出还缺哪些字段？
- 是否需要发布 Chrome Web Store / Edge Add-ons 版本？
